02 —

Node-by-Node Breakdown

NODE ID TYPE PURPOSE
01_INGEST_LeadSource Webhook / Sheets Trigger Polls Google Sheets every 5 min or receives CRM/API webhook POST. Each row = one lead. Entry point of the entire system.
02_INGEST_NormalizeLead Set Maps raw source fields to a canonical lead schema regardless of origin. Ensures uniform structure for all downstream nodes.
03_INGEST_Deduplicate IF Checks lead_id against Redis or Sheets lookup. Stops workflow if already processed. Prevents duplicate sequences.
04_SEQ_InitState Set Bootstraps sequence state: step=0, tag="Cold", sequence_active=true, reply_received=false, last_contact=now.
05_SEQ_SmartScheduler Wait Adaptive delay computed from step number. Step 0→immediate, Step 1→2d, Step 2→4d, Step 3→7d. Implements exponential backoff.
06_AI_Personalize OpenAI Generates step-aware personalized outreach message using lead's industry, role, and pain point. Channel-aware length limits.
07_SEND_ChannelRouter Switch Routes on preferred_channel field: email→Gmail/SMTP, linkedin→HTTP Request, whatsapp→Twilio/WA Business API.
08_SEND_Message Gmail / HTTP Sends the message. On success: updates last_contact, step++. On failure: triggers 18_ERR_RetryHandler.
09_WAIT_ReplyWindow Wait Pauses 48–72h. In production mode: uses "Wait for webhook" so execution resumes instantly when a reply webhook fires.
10_IF_ReplyReceived IF Checks reply_text !== null && !== "". True→AI analysis. False→increment step and loop back to scheduler.
11_SEQ_IncrementStep Set Increments step counter. If step >= max_steps (3): marks Cold, disables sequence. Else: re-queues to SmartScheduler.
12_AI_SentimentIntent OpenAI Classifies reply into: intent (interested/not/neutral/question/OOO), sentiment (positive/neutral/negative), suggested_action, and tag.
13_PROC_ParseAIResponse Code (JS) Parses OpenAI JSON response, strips markdown fences, sets structured fields (intent, sentiment, tag, confidence) for routing.
14_ROUTE_IntentRouter Switch Routes on intent: "interested"→MarkQualified, "not_interested"→StopSequence, "neutral"/"question"→AI draft reply.
15_QUAL_MarkQualified Set Sets tag="Hot", status="Qualified", qualified_at=now. Triggers Slack/email notification to sales rep.
16_SEQ_StopSequence Set Sets sequence_active=false, status="Rejected", tag="Rejected". Prevents any further outreach to this lead.
17_LOG_WriteRecord G-Sheets / MongoDB Upserts lead record in master sheet/collection on lead_id. Appends timestamped event to the events log.
18_ERR_RetryHandler Error Trigger + Wait Catches failures from send/API nodes. Retries 3× with 5→15→45 min back-off. After 3 fails: writes to errors sheet + Slack alert.
19_COLD_ReEngageTrigger Cron Runs every 24h. Queries for tag="Cold" AND days_since_contact >= 14. Re-injects leads to sequence at step=0 with re-engagement flag.

Prompt B — Sentiment & Intent Detection
Classifies inbound replies and returns structured JSON for routing
12_AI_SentimentIntent
You are an AI trained to analyze B2B sales replies. Incoming reply from {{ $json.first_name }} at {{ $json.company }}: """ {{ $json.reply_text }} """ Classify this reply and return ONLY valid JSON (no markdown, no explanation): { "intent": "interested" | "not_interested" | "neutral" | "question" | "out_of_office", "sentiment": "positive" | "neutral" | "negative", "confidence": 0.0–1.0, "tag": "Hot" | "Warm" | "Cold" | "Rejected", "suggested_action": "schedule_call" | "send_followup" | "stop_sequence" | "answer_question" | "retry_later", "summary": "one sentence describing the reply intent" } Rules: - "interested" → asks for demo, call, more info, or expresses positive curiosity - "not_interested" → says no, unsubscribe, stop, or remove me - "out_of_office" → OOO auto-response (set suggested_action: retry_later) - "question" → asks something without committing - Tag mapping: interested→Hot, question→Warm, neutral→Cold, not_interested→Rejected
13_PROC_ParseAIResponse — Code Node (JavaScript) JS 
const raw = $json.choices[0].message.content;
const clean = raw.replace(/```json|```/g, '').trim();

let ai;
try {
  ai = JSON.parse(clean);
} catch (e) {
  throw new Error('AI response parse failed: ' + raw);
}

return {
  intent:     ai.intent     || 'neutral',
  sentiment:  ai.sentiment  || 'neutral',
  tag:        ai.tag        || 'Cold',
  confidence: ai.confidence || 0.5,
  suggested_action: ai.suggested_action,
  ai_summary: ai.summary    || '',
};

Error Handling & Retry Strategy

⚡
Retry Logic
  • All HTTP / send nodes wrapped by Error Trigger node
  • 3 retry attempts on failure
  • Exponential back-off: 5 min → 15 min → 45 min
  • Retry state stored in lead object to survive restarts
  • Idempotency check before every send attempt
🛡️
Idempotency Guards
  • Check sequence_active = true before send
  • Check reply_received = false before send
  • Timestamp lock: skip if last_contact within 30 min
  • Dedup check on lead_id at workflow entry
  • Upsert (not insert) on all log writes
📋
Dead-Letter Queue
  • After 3 failures: write to _errors sheet tab
  • Fields: lead_id, node_name, error_message, timestamp
  • Slack alert to #outreach-errors channel
  • Manual review queue for human intervention
  • Daily error digest via scheduled summary node
18_ERR_RetryHandler — Code Node JS 
const lead = $json.lead;
const retries = lead.retry_count || 0;
const maxRetries = 3;
const delays = [5, 15, 45]; // minutes

if (retries >= maxRetries) {
  // Write to dead-letter sheet + alert Slack
  return { action: 'dead_letter', lead_id: lead.lead_id };
}

return {
  ...lead,
  retry_count: retries + 1,
  next_retry_delay: delays[retries],
  action: 'retry'
};

Scaling Improvements

01 —
Redis Job Queue
Add a Redis node as a job queue in front of the Smart Scheduler. Prevents thundering-herd when thousands of leads enter simultaneously. Enables rate-limited, ordered processing.
Infrastructure
02 —
Sub-workflow Architecture
Split into two n8n workflows: Outreach Master (ingest + init) and Sequence Worker (send + analyze). Call the worker via Execute Workflow node for parallel execution across hundreds of leads.
Architecture
03 —
A/B Message Testing
Add an IF node after personalization that routes 50/50 to two OpenAI prompts (Variant A vs B). Log the variant field in the events sheet to measure and optimize reply rates over time.
Optimization
04 —
Webhook-based Reply Detection
Replace polling with Nylas or Postmark inbound webhooks. Makes reply detection real-time (seconds vs hours). The Wait node switches to "Resume on webhook" mode for instant triggering.
Real-time
05 —
Lead Scoring Engine
Add a Code node after intent routing that calculates a numeric score based on reply_speed + positive_sentiment + step_number. Auto-escalate leads above threshold score to a human rep via Slack.
Intelligence
06 —
Multi-language Support
Add a language field to the lead schema and pass it to the personalization prompt. Claude or GPT-4 handles translation natively — one workflow architecture, global outreach coverage.
Globalization
07 —
Rate Limiting
Add a Wait node (1–2 seconds) between each send to avoid Gmail/LinkedIn daily limits. For Gmail, stay under 500 emails/day on standard accounts. Use a counter in Redis to track daily volume.
Compliance
08 —
Analytics Dashboard
Connect the events collection to Metabase or Retool for a live dashboard showing: reply rates per step, tag distribution, sequence completion %, and channel performance — all auto-updating.
Observability
